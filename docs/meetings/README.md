# Meeting Notes & Decisions Log

Every working session — kick-off, on-site discovery, role-play validation, sign-off — gets a file here.

## File naming

`YYYY-MM-DD-short-kebab-topic.md`

Examples:
- `2026-05-18-kickoff.md`
- `2026-05-22-sales-onsite.md`
- `2026-06-03-finance-roleplay-invoice.md`

## Template

Each file should contain:

1. **Date / location / attendees**
2. **Agenda**
3. **Notes** — bullet form is fine here, this is a working document.
4. **Decisions** — explicit, numbered, with owner and date.
5. **Action items** — with owner and due date.
6. **Open questions raised** — to be answered before the next session.
7. **Artefacts updated** — list of files in this repo that changed as a result of this meeting.

See [`2026-05-18-kickoff.md`](2026-05-18-kickoff.md) for the worked example.
